# AI Knowledge Base

AIに関連するツール、MCP（Model Context Protocol）サーバー、活用スキル、プロンプトテンプレート、および開発アーキテクチャの知見を蓄積・管理するためのリポジトリです。

本リポジトリは、Slackやチャットツールからの入力を基に、AIエージェントが自動的に分類・作成・更新を行う運用を前提として設計されています。情報の陳腐化を防ぐため、詳細な設定手順ではなく「概要」と「一次情報（公式リンク）」の管理に特化しています。

## リポジトリ構成

本リポジトリは以下のディレクトリおよびファイル構造でデータを管理します。

```text
.
├── README.md                 # 本ファイル（全体のインデックスと運用定義）
├── tools/
│   ├── cli-tools.md          # AI関連のCLIツール・ローカル環境構築ツール
│   └── mcp-registry.md       # MCPサーバー・関連プラグイン
├── skills/
│   ├── general-skills.md     # 各種AIサービス、フレームワーク、ライブラリのリンク集
│   └── prompt-templates.md   # プロンプトテンプレートおよびテクニック
└── engineering/
    └── architecture.md       # 開発設計、RAGアーキテクチャ、セキュリティ、他社事例

```

## 記述フォーマット

情報はコンテンツの性質に応じて、以下の2つのパターンのいずれかで記録されます。

### パターンA：リンク・ツール型（Tools, Skills）

1行の箇条書きで構成され、公式ドキュメントへのリンク、最小限の概要、ユーザーのコンテキスト（動機や気づき）、検索用ハッシュタグを含みます。

```markdown
* [{名称}]({URL}) - {概要（1〜2文）}。{ユーザーの動機/コンテキスト/気づき}。 #{タグ1} #{タグ2}

```

### パターンB：プロンプト型（Prompt Templates）

文章量が多くなるプロンプトやテンプレートは、見出しを用いて構造化し、テキストブロック内に全文を格納します。

```markdown
### {プロンプトの名称/用途}
* **リンク/出典:** [{出典名}]({URL})
* **動機/気づき:** {ユーザーの動機やコメント}
* **タグ:** #{タグ1} #{タグ2}
* **プロンプト内容:**
```text
  {プロンプトの全文}

```

## 運用・更新フロー

本リポジトリへのデータ追加は、以下の自動化ワークフローによって実施されます。

1. **インプットの送信**
   ユーザーがSlackまたはチャットツール経由で、AI関連のURL、プロンプト、またはメモを送信します。この際、利用動機や気づきをコメントとして同時に送信することが可能です。
2. **AIによる解析と分類**
   連携されたAIエージェントが投稿内容を解析し、「URL」「概要」「ユーザーのコメント」を抽出します。内容に応じた技術タグ（例: `#mcp`, `#cli`, `#prompt`）を2〜3個自動生成します。
3. **リポジトリの自動更新**
   AIエージェントは定義された「分類ルール」に基づき、対象のMarkdownファイルへ定義フォーマットに従って情報を追記（コミット）します。すでに同一のURLや名称が存在する場合は、重複を避け、ユーザーの「動機/気づき」のみを既存の記述にアップデートします。
